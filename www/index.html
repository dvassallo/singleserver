single server

One VPS. Many apps. Git push to live.

A tiny deploy control plane for people who want to own their infrastructure without babysitting a different runner, secret, and workflow for every small project.

The whole idea

Single Server turns one blank Linux box into a home for many GitHub projects. Each app gets its own Docker container. The server owns deploys. GitHub only sends push events.

git push -> GitHub App sends a signed webhook
apps.yml -> Only allowlisted repos can deploy
kamal -> Build, swap, healthcheck, commit status

Target setup flow

Install the host control plane.

curl -fsSL https://singleserver.com/install.sh | sh

Point the server at your control domain.

singleserver init --domain deploy.example.com --email me@example.com

Connect GitHub once.

singleserver github connect

Add a repo and deploy it.

singleserver add me/my-app --host my-app.example.com --deploy

The core deploy loop is running today. The installer, init, and GitHub connect commands show the target setup flow from the working-backwards roadmap.

What you get

Fast deploys

Build on the server, swap containers, healthcheck, and set a commit status. Small apps can deploy in a few seconds.

One central allowlist

apps.yml decides what can deploy. Install the GitHub App broadly without making every repo deployable.

No repo runners

No GitHub Actions runner per project. No repo deploy keys. No pile of repeated deploy secrets.

Containers per app

Every project runs in its own Docker container, but the host still stays simple and inspectable.

Kamal without boilerplate

Single Server generates deploy config from conventions unless an app intentionally ships its own.

Doctor included

Check the daemon, GitHub App access, checkouts, deploy config, last deploys, and public healthchecks.

The app contract

Keep repositories boring. A Single Server app only needs a Dockerfile, one web port, and a health endpoint. Node, Bun, static sites, SQLite-backed apps: all fine.

Dockerfile -> Single Server can build it
port 80 -> Default app port, override when needed
/up -> Default healthcheck path
